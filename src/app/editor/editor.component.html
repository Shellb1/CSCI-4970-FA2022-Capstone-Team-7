PDF.js Library

PDF.js is an open-source JavaScript library developed and supported by the Mozilla foundation. It is included standard in Mozilla Firefox, and there also is an extension for Google Chrome. Thus, the supported browsers for this particular application are Firefox and Chrome.

Some of the features that this version of the viewer includes are opening and viewing PDFs fully client-side, annotating PDFs fully client-side, as well as loading annotations already present. In addition, there is a feature to add text to the PDF as well. There are also various search and view features. One could also add to the viewer and have it render PDFs from URLs, however, CORS (cross-origin resource sharing) concerns should be taken into account.

In this version of the viewer, we did not find any mechanism for supporting .XMP files, nor support for images and layers without adding on to the viewer. While these features are supported by the library, and thus it would be possible to implement by extending the viewer, it would be fairly complex to do so.

Ultimately, however, the library does a lot of the heavy working with parsing/rendering PDFs, so it is possible to add a lot of features by utilizing the library's APIs. That, in fact, is how the webviewer is built: it is not actually part of the full library, but rather a demo meant to show what is possible via the library. Another benefit is it is written in pure JavaScript/HTML, and thus can be extended and utilized within any framework/development context.

The library comes with much more features that can be utilized outside of those found in the viewer. For example, the library supports dozens of annotation types not currently found in the viewer, such as lines, squares, circles, highlights, underlines, and much more. Basically, any annotation officially supported by the PDF file specification is supported by the library. However, these features would have to be built into the viewer itself to actually render them.

Unfortunately, we did not find any way to save the annotations separately from the PDF itself. The library computes and adds the annotations directly to the actual file blob that gets downloaded to the client's machine. However, it is theoretically possible to build this feature in outside of the library itself, but it would be complex and require a way to track the annotations outside of the library.

Finally, a key benefit of using PDF.js over other editors is it is free and open-source. However, using the viewer we have embedded in our application would require Buildertrend to build upon it or reskin it in some way in accordance with the licensing on the Github repo, which is linked here.
