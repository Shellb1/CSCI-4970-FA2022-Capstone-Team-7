import { Component, OnInit } from '@angular/core';
import { Router } from '@angular/router';
@Component({
  selector: 'app-editor',
  templateUrl: './editor.component.html',
  styleUrls: ['./editor.component.scss']
})
export class EditorComponent implements OnInit {

  constructor(private router: Router) { }

  ngOnInit(): void {
  }
  goToPage(page: String) {
    this.router.navigateByUrl('/' + page);
  }

}

This is the HTML for the PDF.js page. This includes the viewer itself, which in an iframe on this page
that references the viewer in /assets/web/viewer.html.

This also includes the documentation on the other part of the page that describes the PDF.js library
and the features included with it. 
-->
<body>
    <div style="float:left;">
      <iframe id="pdf-js-viewer" src="../../assets/web/viewer.html?file=" title="webviewer" frameborder="0" width="700"
        height="600"></iframe>
    </div>
    <div style="padding-left:750px; padding-right:20px;">
      <h3 style="text-align:center; font-size: 30px;">PDF.js Library</h3>
      <p style="font-size:15px;">
        PDF.js is an open-source JavaScript library developed and supported by the Mozilla foundation. It is included standard in Mozilla Firefox, 
        and there also is an extension for Google Chrome. Thus, the supported browsers 
        for this particular application are Firefox and Chrome. 
        <br/>
        <br/>
        Some of the features that this version of the viewer includes are opening and viewing PDFs fully client-side, 
        annotating PDFs fully client-side, as well as loading annotations already present. In addition, there is a feature 
        to add text to the PDF as well. There are also various search and view features. One could also add to the viewer and
        have it render PDFs from URLs, however, CORS (cross-origin resource sharing) concerns should be taken into account.  
        <br/>
        <br/>
        In this version of the viewer, we did not find any mechanism for supporting .XMP files, nor support for images and layers without
        adding on to the viewer. While these features are supported 
        by the library, and thus it would be possible to implement by extending the viewer, it would be fairly complex to do so. 
        <br/>
        <br/>
        Ultimately, however, the library does a lot of the heavy working with parsing/rendering PDFs, so it is possible to 
        add a lot of features by utilizing the library's APIs. That, in fact, is how the webviewer is built: it is not actually part
        of the full library, but rather a demo meant to show what is possible via the library. Another benefit is it is written in 
        pure JavaScript/HTML, and thus can be extended and utilized within any framework/development context. 
        <br/>
        <br/>
        The library comes with much more features that can be utilized outside of those found in the viewer. For example, the library supports
        dozens of annotation types not currently found in the viewer, such as lines, squares, circles, highlights, underlines, and much
        more. Basically, any annotation officially supported by the PDF file specification is supported by the library. However, these features
        would have to be built into the viewer itself to actually render them. 
        <br/>
        <br/>
        Unfortunately, we did not find any way to save the annotations separately from the PDF itself. The library computes and adds the annotations
        directly to the actual file blob that gets downloaded to the client's machine. However, it is theoretically possible to build this feature
        in outside of the library itself, but it would be complex and require a way to track the annotations outside of the library. 
  
        <br/>
        <br/>
        Finally, a key benefit of using PDF.js over other editors is it is free and open-source. However, using the viewer we have embedded in our application 
        would require Buildertrend to build upon it or reskin it in some way in accordance with the licensing on the Github repo,
        which is linked <a href="https://github.com/mozilla/pdf.js" target="_blank">here.</a>
      </p>
